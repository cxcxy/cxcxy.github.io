RxSwift学习之路

发表于 2017-08-28

RxSwift是ReactiveX 系列的Swift 版本，如果你之前使用OC，并且使用过RAC的话，相信你对函数式响应编程这个概念并不会陌生。

函数式响应编程

响应式编程是一种面向数据流和变化传播的编程规范，数据更新是相关联的。比如很多时候，在写界面的时候，我们需要通过Action来响应事件，在通过通知来监测键盘高度，通过回调来执行网络请求的响应，通过KVO来监测值的改变，这些都是可以通过RxSwift来轻松解决的，他把我们程序中每一个操作都看作一个事件。通俗点的例子就是观察者模式，Observable和Observer， 其中Observable是可被观察的，Observer是观察者，针对Observable发出的信号，作出相应的处理。就比如我们写字，假设一个学生是一个Observable，然后老师是Observer，老师负责监督学生的啦，学生用笔写下来一个字，然后老师看到这个字，会把这个字的发音读出来，然后做出一系列的回应。又或是，老师监考学生，学生有作弊行为，老师负责揪出来这个学生，然后emmmmmm….

从基本理念开始

1.创建和订阅可观测序列

Rxswift 的核心类就是Observable,它的主要作用，就是把事件转换成信号发给订阅者，然后，订阅者接收到信号，做出相应的处理。在iOS里面典型的就是NSNotification啦～ 一个注册，一个监听。

DisposeBag

DisposeBag其实就相当于iOS中的release，会在适当的时候销毁观察者，类似ARC的retain和release。
在这提一下，平时我们项目中用的disposeBag的时候，都会实例化一个DisposeBag，这样就会很让人不耐烦，当然，在这有一个很好的第三方库NSObject-Rx，内部是为NSObject做了一个延展， 这样就不用每次都写let disposeBag = DisposeBag()了，
直接用的时候，用rx_disposeBag替换。

subscribe

subscribe其实就是Observable 发出来的事件啦，有next事件，error事件，completed事件。再次我们都可以拿到这些事件的回调信息的啦～

of：

of 是创建一个 sequence 能发出很多种事件的信号

from：

from 就是从集合中创建 sequence 例如 数组， 字典 或者set 相当于 把 集合 当成一个信号

create：

create 使我们可以自定义可观察的sequence ， 可以自定义 观察者 观察 被观察者 的方式， 例如 是否 监听 completed 这个操作

repeatElement：

repeatElement 创建一个 sequence 发出特定的事件3次 take() 指定事件的次数

generate：

generate 是创建一个可观察的 sequence ，当初始化条件为 true 的时候， 他就会发出所对应的事件

varialbe：

varialbe 是一个包装箱， 就像一个箱子一样，在使用的时候，会调用 asObservable（)拆箱， 里面的value 是 值 ，他不会发出error 事件， 但是会自动发出 completed 事件。 也看 variable 的作用域

startWith：

startWith 在发出事件之前，先发出某个特定的事件消息。 比如发出事件 2，3 然后，我startWith（“1”），那么就会先发出 1，然后2.3

map：

map 通过传入一个函数闭包， 把原来的 sequence 转变为一个新的 sequece 的操作 下面的操作相当于 每个元素相乘

scan：

scan 就是给一个初始化的数，然后不断的拿前一个结果和最新的值进行处理操作。

filter：

filter 就是过滤掉某些不符合要求的事件。

take：

take 只处理前几个事件信号,
